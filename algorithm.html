Algorithm description of ZEOGS

Zebrafish Expression Ontology of Gene Sets implements a Perl-based workflow to obtain a summary of known gene expression patterns of a gene set. The image below gives you a graphical overview of the ZEOGS algorithm. Details of the input options are described on the Help page. Here we will look at the steps involved in obtaining output results from the input data and options.

Input

The user initially selects which dataset will be used by the program ("Data source"), inputs a list of gene identifiers ("Gene ID sample"). To give the results a temporal context, the user also has to select a stage ("Stage of development"), which will be used to filter anatomical terms known for genes in the list. In addition, anatomical terms will then be tested for over-representation using the hyper-geometric test with Benjamini-Hochberg multiple testing correction and the user can then choose how to sort them ("Sorting option"). Finally, since in many experiments it is not possible to test really all known genes, the user can directly enter a list of genes which were tested ("Tested genes option") and from which the sample of gene identifiers was obtained.

Processing and output

The input gene identifiers are first converted ZFIN IDs, which will be used to query expression data sets. Then the program obtains anatomical terms for the input gene identifiers and either for all known zebrafish genes or for genes input in the "Tested genes" option. During this process, anatomical terms are filtered based on whether they are known or were observed at the selected stage of development. In the next step the gene lists and gene counts for each encountered anatomical term are generated. Further down the workflow, the over-representation of each anatomical term is checked with the hypergeometric statistical test on the gene counts for the sample and total gene lists. Finally, all of these data is output in the form of gene identifier tables for each anatomical term and anatomical term tables of gene counts.

The data source and anatomical term filtering choice

The data source option "anatomical terms on ZFIN gene pages + stage filter" is based on the anatomical terms listed on ZFIN gene summary pages. To limit the anatomical terms only to those known for each stage, a simple stage filter was implemented. Each anatomical term is checked against a list of anatomical terms known for the selected stage and if present is added to the results table. Another data source option "Wild-type expression: manually annotated data for some stages" is a more rigourous option for mining expression data in zebrafish. This option is based on the wild-type expression file on ZFIN, from which anatomical terms were extracted and categorized by stage information available in the file. Since this file is based on direct empirical observations, the user receives directly validated information.
