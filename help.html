1. The concept of ZEOGS software.

Zebrafish has become a well-established model system for understanding developmental mechanisms, fundamental biological processes and a variety of biomedically relevant studies. Moreover, functional genomics has a firm hold on research in this model system leading to generation of a large number of high-throughput datasets each providing information on hundreds to tens of thousands genes, proteins or other molecular entities. Since it is impractical and almost impossible to keep in human memory information on such a high number of molecular entitities, we need tools which aggregate information about them and provide an overview for an overall understanding of what the data mean from a functional and expression pattern perspectives. Gene Ontology, Gene Set Enrichment Analysis, KEGG pathway and Ingenuity pathway analysis tools give us a good opportunity to gain deeper insights into functions of gene sets. Moreover, one should not forget that genes are expressed in highly specific spatial patterns and this has important functional consequences. On the other hand, analyzing gene sets from global analyses of zebrafish embryos or larvae may allow inferring in which tissues these genes are preferentially expressed and therefore may be affected by experimental perturbations. ZEOGS is a tool to summarize expression pattern properties of gene sets in zebrafish and to allow such inferences.

The design of many expression profiling and other high-throughput genomic experiments in zebrafish is often based on isolating material from whole embryos or larvae because of either technical limitations or lack of knowledge where potential differentially regulated genes are expressed. Thus, in the case of expression profiling the spatial information on gene expression is lost. After obtaining lists of gene identifiers it is certainly possible and sometimes desirable to manually annotate what is known about expression patterns of differentially-regulated genes using information stored in the Zebrafish Information Network databases. However, such a task is highly time-consuming and despite giving an impression of the range of patterns, tissues and organs does not provide a good summary and any measure of enrichment of anatomical terms encountered in the gene list.

2. Options for data entry and output.

1. Expression data source

  • anatomical terms on ZFIN gene summary pages + a stage filter

    • This data source option is based on the anatomical terms listed on ZFIN gene summary pages. To limit the anatomical terms only to those known for each stage, a simple stage filter was implemented. Each anatomical term is checked against a list of anatomical terms known for the selected stage and if present is added to the results table.

  • Wild-type expression: manually annotated data for some stages

    • This data source option is a more rigourous option for mining expression data in zebrafish. This option is based on the wild-type expression file on ZFIN, from which anatomical terms were extracted and categorized by stage information available in the file. Since this file is based on direct empirical observations, the user receives directly validated information. The disadvantage is that with this option there is very little information on early and late developmental stages. More specifically, our analysis presented in the figure below showed that the 'wild-type expression' option is most applicable for stages 17-33 (5-60 hours post fertilization). This is because these stages have been most frequently studied. At all other stages of zebrafish development its information content is dramatically lower than that of the 'gene summary' option.

    2. Format of gene identifiers

  • ZFIN ID

    • "ZFIN ID" is the primary identifier used by the ZFIN database, e.g. ZDB-GENE-000524-1.

  • ZFIN symbol

    • "ZFIN symbol" is a short gene name identifier (e.g sox9b) that is used by different databases, thus being a generally applicable identifier for zebrafish genes.

    The usage of other identifiers is not optimal for querying ZFIN expression datasets because of the conversion issues between formats from different databases.

    3. Choice of entry mode for gene identifiers - either as text or as a file

    To provide input to the program, the user can paste the gene identifiers into the text area or upload them as a file.

    4. Gene identifiers - the primary information for querying expression term data sets

    Please enter gene identifiers in one of the two formats. In case you do not have identifiers in one of these two formats, you should convert from the available format to either ZFIN ID or ZFIN symbol using BioMart or another conversion software

    5. File with a list of all gene identifiers tested in your experiment

    The results of anatomical term over-representation analysis will likely depend on the genes tested in an experiment. Most high-throughput experiments to date have not been able to test every gene in the zebrafish genome. Therefore, the list of tested genes may be provided so the program can adjust the 'total genes' list and provide a better assessment of actual enrichment of certain anatomical terms. In addition, this possibility of uploading tested genes may be of great use to experiments using custom-designed microarrays. However, this option is not obligatory and by default the program will use information on all known genes with expression information.

    6. Stage of zebrafish, for which the output will be generated

    This option provides a temporal context for the gene set and should always be selected since otherwise the results may not make much sense. Please be aware that the results using the two data source options may differ a lot due to nature of algorithms used in the program.

    7. Choice from options for sorting of anatomical terms

  • Gene count
  • Uncorrected P-value sorted in an ascending order

    • This option sorts the anatomical terms in the results table according to their values, the smaller values being higher in the table.

  • Fraction of total genes with a specific term

    • Using this option, the user can sort anatomical terms according to the ratio of gene count for an anatomical term in the sample to its total count

    Testing for over-representation of anatomical terms using the hypergeometric test with multiple testing correction

    This combinatorial test assumes random sampling of genes and anatomical terms and gives the probability of obtaining the same or a higher gene count for a certain anatomical term. The following formula is used for this purpose

    .

    N – the total number of genes with associated anatomical terms
    n – the number of genes with a specific anatomical term
    m – the number of selected genes
    k – the number of selected genes with the specific anatomical term
    x – a variable for P-value calculation

    ZEOGS additionally adjusts the P-values for multiple corrections, which makes P-values larger and therefore reduces their apparent significance. The mathematics for the correction can be found in the original Benjamini&Hochberg, 1995 paper.


    3. The structure of ZEOGS output.

    Files generated by the program

    ZEOGS generates three files from the input gene identifier list. All of these files will be stored in a newly generated random folder, which will be deleted after 24 hours, so the links to the files can be used for access to the results for one day. The file links are illustrated in the figure below.

    File 1 contains a table of ranked terms identical to the one on the HTML version of the output page and sorted in the same order.

    File 2 contains tables of gene identifiers for each of the anatomical terms encountered for the input gene identifiers. This file can be useful for some downstream analyses focusing on genes in the specific anatomical terms.

    File 3 contains a table of gene identifiers which did not have any anatomical terms associated with them in the current data sets. Thus, if your aim is to quickly filter the genes with unknown expression patterns and then do some experiments on them, you can use the list for this purpose.

    Ranking table of the anatomical terms and summary statistics

    This table is the central piece of ZEOGS output. Each anatomical term in the table is hyper-linked to its page on the ZFIN for additional explanation and hierarchy relationship information. Additionally, there is information on the number of genes with the anatomical term in the sample and for all of the genes with expression information. The next columns contain the P-values calculated using the hyper-geometric test, the Benjamini-Hochberg corrected P-value and the conclusion whether the corrected P-value is below the 0.1 threshold. In addition, table rows are colour coded, orange being the colour for anatomical terms with corrected P-values < 0.1, green labeling the rows for terms with enrichment P-values < 0.1 and all other terms having a neutral background colour.
